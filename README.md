RESUMO DO PROJETO

  Este projeto, desenvolvido em C++ no Visual Studio, simula uma tela composta por 32 pixels organizados em quatro linhas de oito bits cada. Cada pixel representa um bit, podendo estar ligado ou desligado, e é exibido no terminal usando caracteres gráficos e códigos ANSI para cor. O programa inicia exibindo todos os pixels acesos e, após um breve intervalo, entra em um modo automático de teste, no qual são exibidos dez padrões diferentes gerados aleatoriamente a partir de um número inteiro de 32 bits. Concluída essa fase, o usuário passa a controlar a tela através de comandos digitados no teclado, podendo ligar ou desligar bits individualmente, alterar toda a linha, atualizar uma linha da tela ou finalizar o programa. Toda modificação é refletida visualmente no console.
 
  Ao encerrar o modo interativo, o programa entra em uma etapa final temática chamada "Hackeamento", onde o valor inteiro de 32 bits que representa a tela é exibido em formato binário utilizando a biblioteca bitset. Esse valor também é dividido em duas partes de 16 bits, e cada metade da tela é redesenhada com um estilo personalizado, simulando uma invasão do sistema. Para organizar o código, o projeto é estruturado em bibliotecas separadas responsáveis pela manipulação de bits, exibição da tela e geração de números pseudoaleatórios. O trabalho explora conceitos fundamentais como operações bitwise, modularização, entrada de comandos, exibição visual no terminal e manipulação de números binários.

ABSTRACT

  This project, developed in C++ using Visual Studio, simulates a screen composed of 32 pixels arranged in four rows of eight bits each. Each pixel represents a bit, which can be on or off, and is displayed in the terminal using graphic characters and ANSI color codes. The program starts by displaying all lit pixels and, after a short interval, enters in an automatic test mode, in which ten different patterns are displayed, randomly generated from a 32-bit integer. Once this phase is complete, the user can control the screen through commands typed on the keyboard, being able to turn bits on or off individually, change the entire row, update a line on the screen, or terminate the program. All modifications are visually reflected in the console.

  Upon ending the interactive mode, the program enters a final thematic stage called "Hacking," where the 32-bit integer value representing the screen is displayed in binary format using the bitset library. This value is also divided into two 16-bit parts, and each half of the screen is redrawn with a custom style, simulating a system intrusion. To organize the code, the project is structured into separate libraries responsible for bit manipulation, screen display, and pseudo-random number generation. The work explores fundamental concepts such as bitwise operations, modularization, command input, visual display in the terminal, and binary number manipulation.
